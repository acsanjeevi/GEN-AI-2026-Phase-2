 1. EXECUTIVE SUMMARY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Our quality engineering organisation currently operates a Cypress + TypeScript
test automation framework with MochaAwesome reporting. While functional, this
approach is constrained by manual test authoring, slow migration timelines, and
an absence of AI-driven intelligence across the QA lifecycle.

This Business Case proposes the development of a Single AI Agentic Tool —
a LangChain-orchestrated, RAG-powered, Deep Eval-governed system — that
automates the entire QA pipeline: from business requirements to deployed
Playwright tests running in Azure DevOps CI/CD pipelines.

The proposed solution delivers an estimated 75–85% reduction in QA engineering
effort per sprint, eliminates weeks of manual migration work, and establishes
an intelligent, self-improving quality assurance capability for the enterprise.

This initiative is proposed for development at the upcoming Hackathon,
with a working prototype demonstrable within a single-day sprint.

2.1  The Core Problem
─────────────────────
The QA Engineering team currently operates with two fundamental inefficiencies
that directly impact velocity, quality, and cost:

  PROBLEM 1 — Manual Test Creation
  Every test case is authored manually from business requirements.
  This is time-consuming, error-prone, and creates inconsistency across
  test suites authored by different engineers.

  PROBLEM 2 — Framework Migration Bottleneck
  The industry has moved decisively to Playwright + TypeScript as the
  standard for modern web test automation. Our Cypress-based framework is
  falling behind. A full manual migration is estimated at 3–4 weeks of
  dedicated engineering effort, delaying new feature delivery.

2.2  Impact of Current State
─────────────────────────────
  - 40+ hours per sprint per engineer consumed by manual test authoring
  - 3–4 weeks of migration effort consuming 2 FTE capacity
  - No intelligent analysis of test coverage gaps
  - No automated quality gate on AI-generated or newly authored tests
  - Manual Azure DevOps YAML authoring required for every new pipeline
  - MochaAwesome HTML reports not integrated with Azure DevOps Test Plans

4.1  Solution Overview
───────────────────────
Build a Single AI Agentic Tool — accessible via one unified entry point —
that orchestrates five specialised AI agents across the complete QA lifecycle.

  Tool Name      : QA-AIAT (QA AI Agentic Tool)
  Architecture   : LangChain multi-agent orchestration
  Knowledge Base : RAG (vector store of requirements, docs, templates)
  Quality Gate   : Deep Eval (LLM output scoring and hallucination detection)

4.2  Migration Target State
────────────────────────────
  FROM : Cypress + TypeScript + MochaAwesome Report
  TO   : Playwright + TypeScript + YAML Config + Azure DevOps CI/CD Pipeline

4.3  AI Component Roles
────────────────────────
  Component     | Role in QA-AIAT
  ─────────────────────────────────────────────────────────────────────────────
  RAG           | Ingests and retrieves from requirement documents.
                | Ensures AI-generated content is grounded in YOUR project
                | knowledge, not generic LLM hallucinations.
  ─────────────────────────────────────────────────────────────────────────────
  LangChain     | Orchestrates the sequence of agent tasks. Routes inputs
                | to the correct agent, maintains workflow memory, and
                | manages inter-agent communication.
  ─────────────────────────────────────────────────────────────────────────────
  Deep Eval     | Acts as an automated quality gate. Scores every AI-generated
                | test for accuracy, hallucination, relevance, and completeness
                | before it is committed to the repository.
  ─────────────────────────────────────────────────────────────────────────────

4.4  Five Specialised Agents
──────────────────────────────
  Agent 1 — Requirement Agent  : Parses requirements via RAG; extracts scenarios
  Agent 2 — Test Design Agent  : Generates Playwright TypeScript tests (LangChain)
  Agent 3 — Migration Agent    : Converts Cypress tests to Playwright (LangChain)
  Agent 4 — Evaluation Agent   : Scores test quality via Deep Eval
  Agent 5 — Pipeline Agent     : Generates Azure DevOps YAML pipelines (LangChain)
